# Test Documentation

Use this folder to store test reports from real bench sessions and full-arm validation.

## Recommended Workflow

1. Copy `docs/tests/TEST_REPORT_TEMPLATE.md`
2. Save it as `docs/tests/YYYY-MM-DD_short-test-name.md`
3. Fill in the setup, procedure, results, and next actions
4. Link photos, videos, or serial logs when available

## Suggested First Reports

- one MG996R bench test
- one SG90 bench test
- one Nunchuck I2C stability session
- one full-arm 10-minute stress test
